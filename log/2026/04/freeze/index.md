# 🧊 freeze/melt in 2026

In an endless quest to create test data for [qemount](/dev/rust/qemount), I
stumbled on the source code for 90s archival tool `freeze`. I hacked around with
it and got it working in Linux, but it didn't come with a license so I was a bit
worried about actually using it.

But it turns out Leo Broukhis (of the Underhanded C Contest) never stopped
writing code, is active on GitHub, and cheerfully 
[responds to pings](https://github.com/bitplane/freeze/issues/1).

So we now have `freeze` / `melt` on modern systems, and they're fully MIT
licensed.

Thanks, Leo!

* [🐱 source](https://github.com/bitplane/freeze)
